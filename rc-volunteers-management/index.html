Quick help

This help section is built directly into the tool and stays lightweight on purpose: mostly static HTML, no heavy extra logic, no large libraries, and no background processing.

What the tool does

The tool keeps one central volunteer master list for the Regional Convention. Every helper can be assigned to one congregation, marked as brother or sister, reviewed for standing, and ticked for one or several departments.

Main strength

You do not need duplicate entries. One helper can be used for multiple departments simply by ticking several department boxes in the same volunteer record.

Templates to congregations

The message template section prepares congregation correspondence almost completely for you.

  • The congregation email is shown directly in the preview area.
  • The helper list for the selected congregation is inserted automatically.
  • You can copy the prepared text instead of writing every message manually.
  • The placeholders [EVENT_NAME], [EVENT_DATE], [CONGREGATION], [VOLUNTEER_LIST], and [SENDER_SIGNATURE] are replaced automatically in the preview.

In practice that means the mail is already largely prepared: recipient context, congregation email, and volunteer list are ready, so you usually only need a quick final check before sending.

Typical workflow for congregation messages
  1. Import or enter volunteers in the master list.
  2. Check whether each helper is linked to the correct congregation.
  3. Open Congregations & messages.
  4. Select or preview a congregation.
  5. Let the tool build the message preview automatically.
  6. Copy the text and use the shown congregation email for sending.
  7. Afterwards, mark the congregation as contacted or replied.
Multiple departments for one helper

A helper can serve in several departments at the same time. Example: the same brother can be marked for Attendants, Parking, and Safety.

  • No duplicate person entries are needed.
  • The department ticks only define where the helper can serve.
  • Department filters help you quickly see all helpers for one area.
Official congregations, extra names, and pool helpers

The official congregation overview is the base for the congregation communication workflow. Extra or unmatched congregation names can still exist for volunteers, especially after imports.

  • If imported helpers cannot be matched cleanly, they can remain in the pool until you assign them manually.
  • This prevents volunteers from getting lost just because the congregation name is not yet perfect.
  • Only the official congregation list is meant to drive the message workflow.
Filters, review, and backups

Use the filters to answer practical questions quickly, for example which congregation already has volunteers, who is marked for Safety, or where standing is still unknown.

  • Export backups regularly, especially after larger imports or many edits.
  • Use congregation follow-up states to keep track of open, contacted, and replied congregations.
  • Bulk rename is useful if one imported congregation name needs to be unified everywhere.
